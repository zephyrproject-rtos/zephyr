Logging in multi-domain environment
###################################

This code sample shows how to use the multidomain logging feature in Zephyr.

Overview
********

This sample utilizes a HOST/REMOTE architecture where the primary core (HOST) aggregates and processes log messages generated by the secondary cores (REMOTE) via the IPC service.
It utilizes the Inter-Processor Communication (IPC) service to route log messages between domains, allowing you to collect and view logs from multiple cores in a single terminal output.

Requirements
************
The sample requires a board with multiple cores and IPC support (such as the nRF5340 DK).It relies on the following key configuration options being enabled:

* ``CONFIG_LOG_MULTIDOMAIN``
* ``CONFIG_IPC_SERVICE``
* ``CONFIG_LOG_LINK_IPC_SERVICE``

Building and Running
********************
This sample can be found under ``samples/subsys/logging/multidomain`` in the Zephyr source tree.

You can build and run this sample on the nRF5340 DK using the following commands:

.. zephyr-app-commands::
   :zephyr-app: samples/subsys/logging/multidomain
   :board: nrf5340dk/nrf5340/cpuapp
   :compact:
   :goals: build flash
   :west-args: --sysbuild
